# 📘 راهنمای جامع استقرار و پیکربندی سامانه (ویژه واحد IT)
### سامانه یکپارچه نظارت، ارزیابی و مدیریت کیفیت تامین‌کنندگان مواد اولیه دارویی (API)

این مستند فنی با هدف تسهیل فرآیند پیکربندی، راه‌اندازی و تحویل نهایی سامانه به سرورهای عملیاتی واحد IT شرکت تدوین گردیده است. این سامانه به‌صورت **Full-Stack (React 19 + Express + Prisma ORM)** طراحی شده و مجهز به موتور هوشمند سوئیچ بین پایگاه‌داده موقت محلی (JSON Storage) و پایگاه‌داده رابطه‌ای پایدار سروری (PostgreSQL) است.

---

## 🏗️ ۱. معماری و ساختار فنی سامانه

*   **لایه فرانت‌اند (Frontend):** کتابخانه React 19 به همراه ابزار بیلد Vite، سیستم طراحی پیشرفته Tailwind CSS و انیمیشن‌های تعاملی Motion.
*   **لایه بک‌اند (Backend):** وب‌سرور سبک و پرسرعت Express هماهنگ با فرانت‌اند بر روی پورت `3000`.
*   **لایه دسترسی به داده (ORM):** نرم‌افزار قدرتمند Prisma ORM جهت برقراری ارتباط کاملاً ایمن و بهینه با سرور پایگاه‌داده رابطه‌ای.
*   **کامپایل بهینه:** فرآیند بیلد سامانه (`npm run build`) کدهای سمت بک‌اند را در قالب یک فایل کامپایل‌شده واحد و سبک به نام `dist/server.cjs` تجمیع و بسته‌بندی می‌کند تا سرعت بارگذاری اولیه کانتینرها در محیط تولیدی به حداکثر برسد.
*   **سیستم هوشمند تاب‌آوری (Self-Healing Storage):** بک‌اند سامانه مجهز به سیستم هوشمند خطایابی است؛ در صورت عدم دسترسی یا اختلال در شبکه ارتباطی دیتابیس اصلی PostgreSQL، نرم‌افزار به‌طور خودکار و بدون ایجاد دان‌تایم (Downtime) به دیتابیس محلی موجود در مسیر `database/vendors.json` مراجعت کرده و پایداری سرویس را حفظ می‌کند.

---

## 🔐 ۲. ساختار نقش‌ها و کنترل دسترسی بر اساس سطح مسئولیت (RBAC)

پیکربندی امنیتی سامانه دارای ۵ نقش سازمانی کلیدی با دسترسی‌های کاملاً مجزا و کنترل‌شده در لایه بک‌اند است:

| نقش (Role) | عنوان کاربری | سطح دسترسی و مسئولیت‌ها | کلمه عبور پیش‌فرض |
| :--- | :--- | :--- | :--- |
| **`admin`** | مدیر ارشد سیستم | دسترسی مطلق به تمام ارزیابی‌ها، ویرایش فرم‌ها، دانلود فایل‌های پشتیبان (JSON)، دریافت گزارش‌های خروجی اکسل (XLSX)، ثبت سورس جدید و مدیریت کامل لاگ‌ها | `123456` |
| **`commercial`** | واحد بازرگانی | مشاهده کارنامه‌ها و ثبت/ویرایش تخصصی امتیازات و معیارهای تجاری تامین‌کنندگان | `123` |
| **`qa`** | واحد تضمین کیفیت (QA) | مشاهده کارنامه‌ها، ثبت/ویرایش تخصصی امتیازات فنی-کیفی آزمایشگاهی و مدیریت وضعیت نهایی تایید سورس‌ها | `123` |
| **`planning`** | واحد برنامه‌ریزی و انبار | مشاهده کارنامه‌ها و ثبت/ویرایش امتیازات مربوط به تدارکات، زنجیره تامین و انبارداری | `123` |
| **`finance`** | واحد مالی | مشاهده کارنامه‌ها و ثبت/ویرایش امتیازات مربوط به شاخص‌های اعتباری و مالی تامین‌کنندگان | `123` |

> 🔒 **الزام رعایت امنیت سایبری:** سامانه مجهز به لایه رمزنگاری یک‌طرفه با الگوریتم **PBKDF2 + SHA512** به همراه سیستم نمک‌زنی هوشمند (Salt) است. در اولین ورود هر یک از نقش‌های کاربری فوق، سامانه به‌صورت اجباری کاربر را به فرم **تغییر رمز عبور پیش‌فرض** هدایت می‌کند تا امنیت سامانه تضمین گردد.

## 🛠️ ۳. پیش‌نیازهای سخت‌افزاری و نرم‌افزاری سرور میزبان

جهت اجرای بی‌نقص سامانه بر روی سرورهای داخلی شرکت، مطمئن شوید ابزارهای زیر نصب و آماده به کار هستند:
1. **Node.js:** نسخه `18.x` یا بالاتر (نسخه پیشنهادی: نسخه LTS Node.js v20)
2. **PostgreSQL Server:** نسخه `13.x` یا بالاتر با دسترسی کامل ایجاد جداول و ارتباط شبکه باز روی پورت `5432`.

---

## 🚀 ۴. مراحل گام‌به‌گام راه‌اندازی و استقرار (سریع)

### روش اول: استقرار کانتینری با Docker Compose (توصیه‌شده برای سرور شرکت)

کدهای سامانه مجهز به ساختار بهینه‌سازی شده برای Docker هستند. با اجرای دستور زیر در مسیر اصلی پروژه، پایگاه‌داده PostgreSQL و وب‌سرور سامانه به‌طور هم‌زمان، کامپایل، کانفیگ و بر روی پورت `3000` سرور بالا خواهند آمد:

```bash
docker compose up -d --build
```

---

### روش دوم: استقرار دستی (Manual Deployment)

#### گام ۱: نصب بسته‌های وابستگی (Dependencies)
```bash
npm install
```

#### گام ۲: تنظیم متغیرهای محیطی (Environment Variables)
یک فایل به نام `.env` در ریشه اصلی پروژه ایجاد کرده و مقادیر زیر را مطابق با سرور دیتابیس و تنظیمات امنیت شبکه خود شخصی‌سازی کنید:

```env
# آدرس اتصال پایدار به پایگاه‌داده PostgreSQL واحد IT
DATABASE_URL="postgresql://pharma_admin:SecurePassword123@your-db-host:5432/pharma_regulatory_db?schema=public"

# کلید اختصاصی و محرمانه سرور جهت امضا و صحت‌سنجی توکن‌های ورود کاربران (JWT)
JWT_SECRET="PharmaRegRegulatorySecretKey2026Enterprise"
```

#### گام ۳: ایجاد خودکار ساختار جداول پایگاه‌داده (Database Migration)
برای ایجاد خودکار و منسجم تمامی جداول و کلیدهای خارجی در دیتابیس PostgreSQL، دستور زیر را اجرا کنید:
```bash
npx prisma db push
```

#### گام ۴: تزریق داده‌های پیش‌فرض کیفی و دارویی (Data Seeding)
جهت انتقال اطلاعات اولیه تامین‌کنندگان، کاتالوگ مواد مرجع (با مشخصات CAS و IRC) و نقش‌های کاربری پایه به پایگاه‌داده PostgreSQL:
```bash
npm run seed
```

#### گام ۵: بیلد و کامپایل نسخه تولیدی (Production Build)
سمت کلاینت و سرور را به صورت فشرده و بهینه شده کمپایل کنید:
```bash
npm run build
```

#### گام ۶: اجرای نهایی سرویس بر روی پورت ۳۰۰۰
```bash
npm run start
```

> 💡 **پیشنهاد پایداری سرویس (Keep-Alive):** توصیه می‌شود برای اجرای پس‌زمینه مداوم و خودکار در لینوکس سرور از مدیر فرآیند `pm2` استفاده فرمایید:
> ```bash
> pm2 start dist/server.cjs --name "pharma-regulatory-app" --update-env
> ```

## 🩺 ۵. مانیتورینگ و بررسی سلامت سرویس (Health Check)

سامانه دارای یک اندپوینت مانیتورینگ اختصاصی برای بررسی سلامت پایگاه‌داده و وب‌سرور است:
*   **آدرس وب‌سرویس:** `http://localhost:3000/api/health`
*   این اندپوینت یک خروجی استاندارد JSON حاوی وضعیت کلی سیستم (`status: "ok"`) و زمان فعلی سرور را برمی‌گرداند که جهت مانیتورینگ خودکار توسط ابزارهای شبکه (همچون Zabbix یا PRTG) قابل استفاده است.

---

## 📂 ۶. ساختار فیزیکی پایگاه‌داده رابطه‌ای (Entity-Relationship)

پایگاه‌داده عملیاتی سامانه در بستر PostgreSQL از ۵ جدول اصلی تشکیل شده است:
1.  **`vendors` (جدول تامین‌کنندگان):** نگهداری پروفایل، گرید محاسباتی، سطح ریسک و فاز ارزیابی کمپانی‌ها.
2.  **`materials` (کاتالوگ مواد موثره مرجع):** ثبت مشخصات فنی مواد اولیه دارویی از قبیل کدهای CAS و IRC.
3.  **`vendor_materials` (رابطه چندبه‌چند واسط):** طبقه‌بندی هوشمند روابط مواد دارویی با کارخانه‌ها و ردیابی وضعیت نمونه‌های تستی.
4.  **`evaluations` (ارزیابی‌ها و امتیازدهی‌ها):** ثبت سوابق امتیازدهی بخش‌های QA، بازرگانی، مالی و برنامه‌ریزی به همراه دلایل عدم انطباق (NCR/OOS).
5.  **`audit_logs` (رهگیری و زنجیره ممیزی):** ثبت هرگونه اقدام، ویرایش و تایید توسط کاربران برای تامین استانداردهای سخت‌گیرانه صنایع دارویی (Regulatory Audit Trail).
